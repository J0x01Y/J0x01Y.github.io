發表於 2022-01-11 更新於 2022-01-19 分類於 OpenBmc

What is FRU (Field Replaceable Unit) Information Device

在IPMI規範中是使用FRU儲存 “Inventory” 的資料並且可透過Ipmitool將規範定義的內容讀取並解析, 而FRU Information Device主要是一個實際的EEPROM Chip, 但亦可能是裝置本身就具備FRU的資訊.

FRU Information Device的規範中已定義通用的格式, 資料存放方式如下表

Size(bytes) Name Description
8 Common Header 描述其他區塊的偏移量
72 Intternal Use Area 保留給BMC及主板自定義區塊
32 Chassis Info Area 主機資訊, 其具有唯一性整個系統中只有一個FRU裝置具備這個資訊
54 Board Info Area 主板資訊
80* Product Info Area 產品資訊

*規範中建議最少預留80 bytes以上
因為各區的起始位址是由Common Header定義故每個區塊的大小可以自行調整, 上述只是規範中建議的空間分配

Common Header

Byte Description
0 Common Header Version, 目前只有定義0x01
1 Internal Use data offset
2 Chassis data offset
3 Board data offset
4 Product data offset
5 MultiRecord data offset
6 0x00
7 檢查碼

需要特別注意FRU資料是以8 byte對齊故每個offset的數值還需要乘8才是實際的偏移量

Chassis Info

Length Description
1 Format Version
1 Area Length
1 Chassis Type
N Part Number
N Serial Number
N OEM Define
N 0x00 對齊位元
1 Checksum

Board Info

Length Description
1 Format Version
1 Area Length
1 Language Code
3 Mfg. Data/ Time, 從1996/1/1 0:00開始算的多少分鐘
N Maufactuer Name
N Product Name
N Serial Number
N Part Number
N FRU File ID
N OEM Define
1 0xC1 代表資料結尾
N 0x00 對齊位元
1 Checksum

Product Info

Length Description
1 Format Version
1 Area Length
1 Language Code
N Maufactuer Name
N Product Name
N Part/Model Number
N Product Version
N Serial Number
N Asset tag
N FRU File ID
N OEM Define
1 0xC1 代表資料結尾
N 0x00 對齊位元
1 Checksum

phosphor-ipmi-fru

主要功能是解析EEPROM的資料並依照設定檔的定義將其的更新到Inverntory service, 讓其他Module可以存取FRU的資料
Recipe meta-phosphor/recipes-phosphor/ipmi/phosphor-ipmi-fru_git.bb
Source https://github.com/openbmc/ipmi-fru-parser

需要特別注意的是外部只有Ipmi可以修改fru data, ipmi wetie fru cmd實作是透過phosphor-ipmi-fru提供的libstrgfnhandler.so裡面會註冊Net/Cmd = 0x0A/0x12, 但是只有將傳入的Fru binary更新到Dbus沒有寫回實際的EEPROM, 所以重啟後會看到Inventory會被還原回實際EEPROM的資料.

實際上phosphor-ipmi-fru是提供一個onesot service, 透過of-name-to-eeprom.sh讀取eeprom並將解析後的資料丟到d-bus

#!/bin/bash -eu
[ -n "${OF_NAME+1}" ] || exit 0
path="$(grep -xl "$OF_NAME" /sys/bus/i2c/devices/*/of_node/name)"
eeprom="${path%/of_node/name}/eeprom"
sed -i "s,^SYSFS_PATH=.*$,SYSFS_PATH=$eeprom," "$1"

需要帶入兩個參數path & eeprom需要在自己的layer加入對應的檔案內容如下
meta-custom/recipes-phosphor/ipmi/phosphor-ipmi-fru/obmc/eeproms/*

SYSFS_PATH=/sys/bus/i2c/devices/6-0054/eeprom
FRUID=0

Git repository搬移到gerrit

發表於 2021-12-20 更新於 2022-01-11 分類於 Git Server

Mirror repo form gitlab to gerrit

主要是需要加上--mirror參數再進行clone/push

git clone --mirror ssh://gitlab/repo.git
cd repo.git/
git remote add gerrit ssh://gerrit/repo
git push --mirror gerrit

但push時會產生許多錯誤, 需要針對Gerrit server進行調整, 以下是我遇到的幾個錯誤以及解決方式

remote: You need 'Forge Committer' rights to push commits with another user as committer

因為TAG創建者不是推送者, 就會顯示這個錯誤; git push時會顯示以下內容

git push --mirror gerrit
remote: You need 'Forge Committer' rights to push commits with another user as committer.
remote: User: J1Y
remote: Contact an administrator to fix the permissions
! [remote rejected]     2.10.0-dev -> 2.10.0-dev (prohibited by Gerrit: not permitted: forge committer)
! [remote rejected]     2.10.0-rc1 -> 2.10.0-rc1 (prohibited by Gerrit: not permitted: forge committer)

需要修改Gerrit權限設定開放以下權限
Allow Forge Author Identity
Allow Forge Committer Identity
Allow Forge Server Identity

remote: commit 028af1d: warning: subject >200 characters; use shorter first paragraph

開源的專案的commit message裡面通常會包含許多討論內容, 推送時會將其刪減需要修改gerrit的設定,設定檔位於gerrit-site/etc/gerrit.config, 根據需求調整commitmessage.maxSubjectLength & commitmessage.maxLineLength同時建議也修改receive.maxBatchCommits & receive.timeout這兩個設定會影響一次推送的commit數量限制以及超時上限

[receive]
	maxBatchCommits = 1000000
	timeout = 120min
[commitmessage]
	maxSubjectLength = 200
	maxLineLength = 200
remote: Processing changes: refs: 11 (-)error: branches refs/heads/master, refs/meta/config:

這個是因為推送時遠端已有相同的分支, 可以將Gerrit的master刪除並且從Gerrit抓config

git fetch gerrit refs/meta/config:refs/meta/config
! [remote rejected] v0.6.1 -> v0.6.1 (prohibited by Gerrit: update for creating new commit object not permitted)

這個我就不知道出現的原因, 唯有這個tag沒辦法推上去. 我又額外開啟兩個權限之後就一切正常
Allow Create Reference
Allow Push with or without force
