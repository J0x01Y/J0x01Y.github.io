發表於 2022-01-11 更新於 2025-04-22 分類於 OpenBmc

What is FRU (Field Replaceable Unit) Information Device

在IPMI規範中是使用FRU儲存 “Inventory” 的資料並且可透過Ipmitool將規範定義的內容讀取並解析, 而FRU Information Device主要是一個實際的EEPROM Chip, 但亦可能是裝置本身就具備FRU的資訊.

FRU Information Device的規範中已定義通用的格式, 資料存放方式如下表

Size(bytes) Name Description
8 Common Header 描述其他區塊的偏移量
72 Intternal Use Area 保留給BMC及主板自定義區塊
32 Chassis Info Area 主機資訊, 其具有唯一性整個系統中只有一個FRU裝置具備這個資訊
54 Board Info Area 主板資訊
80* Product Info Area 產品資訊

*規範中建議最少預留80 bytes以上
因為各區的起始位址是由Common Header定義故每個區塊的大小可以自行調整, 上述只是規範中建議的空間分配

Common Header

Byte Description
0 Common Header Version, 目前只有定義0x01
1 Internal Use data offset
2 Chassis data offset
3 Board data offset
4 Product data offset
5 MultiRecord data offset
6 0x00
7 檢查碼

需要特別注意FRU資料是以8 byte對齊故每個offset的數值還需要乘8才是實際的偏移量

Chassis Info

Length Description
1 Format Version
1 Area Length
1 Chassis Type
N Part Number
N Serial Number
N OEM Define
N 0x00 對齊位元
1 Checksum

Board Info

Length Description
1 Format Version
1 Area Length
1 Language Code
3 Mfg. Data/ Time, 從1996/1/1 0:00開始算的多少分鐘
N Maufactuer Name
N Product Name
N Serial Number
N Part Number
N FRU File ID
N OEM Define
1 0xC1 代表資料結尾
N 0x00 對齊位元
1 Checksum

Product Info

Length Description
1 Format Version
1 Area Length
1 Language Code
N Maufactuer Name
N Product Name
N Part/Model Number
N Product Version
N Serial Number
N Asset tag
N FRU File ID
N OEM Define
1 0xC1 代表資料結尾
N 0x00 對齊位元
1 Checksum

phosphor-ipmi-fru

主要功能是解析EEPROM的資料並依照設定檔的定義將其的更新到Inverntory service, 讓其他Module可以存取FRU的資料
Recipe meta-phosphor/recipes-phosphor/ipmi/phosphor-ipmi-fru_git.bb
Source ipmi-fru-parser

classDiagram

    EEPROM <.. phosphor_ipmi_fru
    phosphor_ipmi_fru o.. InventoryManager
    phosphor_ipmi_fru <-- Ipmid 
    InventoryManager <|.. Ipmid
    InventoryManager <|.. REST_API
    InventoryManager <|.. Redfish
    class EEPROM{
        -uint8_t data[]
        +read()
        +write()
    }
    class phosphor_ipmi_fru{
        -Chassis AreaData
        -Board AreaData
        -Product AreaData
        +exporInverntory()
        +ipmiStorageWriteFruData()
    }
    class InventoryManager{
        +path xyz.openbmc_project.Inventory
        +GetAll()
    }
    class REST_API{
        +getInventory()
    }
    class Redfish{
        +getInventory()
    }
    class Ipmid{
        +getFruData()
        +setFruData()
    }

需要特別注意的是外部只有Ipmi可以修改fru data, ipmi wetie fru cmd實作是透過phosphor-ipmi-fru提供的libstrgfnhandler.so裡面會註冊Net/Cmd = 0x0A/0x12, 但是只有將傳入的Fru binary更新到Dbus沒有寫回實際的EEPROM, 所以重啟後會看到Inventory會被還原回實際EEPROM的資料.

而FRU提供的資料只能透過InventoryManager取得映射後的結果, 例如取得FRU board area的資料就需要去system/chassis/motherboard裡面找, 而與FRU資料定義的關係是透過yaml設定在本文後面會提到
